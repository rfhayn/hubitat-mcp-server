# Insights Log

Technical insights discovered during implementation. Non-obvious observations worth remembering.

| Date | Milestone | Topic | Insight | Verification | Status |
|------|-----------|-------|---------|--------------|--------|
| 2026-03-12 | M1 | Maker API ceiling | Maker API cannot create/modify rules, manage apps/drivers, access diagnostics, or create virtual devices. Only kingpanther13's on-hub Groovy app bypasses this by using internal Hubitat APIs. Our TypeScript server is scoped to what Maker API exposes. | Confirmed via Hubitat docs and community implementations | CONFIRMED |
| 2026-03-12 | M1 | MCP transport for mobile | Claude Code mobile and Claude.ai can ONLY connect to MCP servers via HTTP transport. Stdio (subprocess) only works for local CLI/Desktop. Dual transport (stdio + Streamable HTTP) is required to support "Claude anywhere." | Confirmed via MCP spec and Claude Code docs | CONFIRMED |
| 2026-03-12 | M1 | ngrok Node.js SDK | `@ngrok/ngrok` npm package embeds the tunnel binary directly in the Node.js process. No separate CLI install needed. This enables single-process deployment (MCP server + tunnel) with one systemd service. | Confirmed via ngrok docs | CONFIRMED |
| 2026-03-12 | M1 | Community landscape | 9 community Hubitat MCP repos exist, none official. Most are Python (4), only 2 TypeScript/JS. Max 4 stars. Fragmented, no clear winner. kingpanther13 (Groovy, 69 tools) is most capable but runs on hub and is "99% AI-generated." | GitHub search | CONFIRMED |
| 2026-03-12 | M1 | EventSocket local-only | Hubitat's EventSocket (WebSocket for real-time events) is only accessible on the local network. For remote deployments, webhooks are the only real-time option. | Hubitat docs | CONFIRMED |
| 2026-03-12 | M1.1 | Bulk query needed (M2) | Asking "what doors are open?" triggered 15 individual `get_device` MCP calls — one per contact sensor. Maker API's `GET /devices/all` returns all devices with full attributes in a single call. M2 should add a `get_devices_by_capability` tool that filters this response server-side, returning only matching devices in one round-trip. | Observed during live testing | CONFIRMED |
| 2026-03-12 | M1.1 | .env inline comments break shell | Inline comments in .env files (e.g., `FOO=bar # comment`) work with Node's dotenv but break `source .env` in bash — the comment becomes part of the value. Use comments on their own lines only. | Observed during testing | CONFIRMED |
| 2026-03-12 | M1.1 | CLI transport override needed | When .env sets `MCP_TRANSPORT=http` but Claude Code launches the server as stdio, the server starts an HTTP listener instead of speaking MCP protocol. Added `--stdio`/`--http` CLI flags to override the env var. | Observed during testing | CONFIRMED |
| 2026-03-12 | M1.2 | Systemd/EnvironmentFile | Systemd services don't source .env files or inherit shell environment. `EnvironmentFile=-/path/.env` is required in the `[Service]` section. The `-` prefix makes it non-fatal if missing. Without this, Node's `dotenv/config` may fail to locate `.env` depending on cwd at service start. | Discovered during Pi production audit | CONFIRMED |
| 2026-03-12 | M1.2 | Claude.ai/mobile no auth headers | Claude.ai and Claude mobile connectors do NOT support custom Authorization headers on MCP servers. Only bare URLs or OAuth. Any MCP server accessed from Claude mobile must be authless or use OAuth. Bearer token auth only works with Claude Code CLI (`--header` flag). | Discovered during Pi install — 401 on mobile | CONFIRMED |
| 2026-03-12 | M1.2 | NodeSource for Pi | `sudo apt install nodejs` on Raspberry Pi OS gives Node.js 12-18 depending on OS version. Must use NodeSource (`curl -fsSL https://deb.nodesource.com/setup_22.x`) to get Node 22. README and setup instructions must reference NodeSource, not apt defaults. | Discovered during Pi install | CONFIRMED |
| 2026-03-12 | M1.2 | echo -e for colors | `echo` without `-e` flag prints raw ANSI escape codes as literal text on some Linux systems. Always use `echo -e` when strings contain color variables (`$GREEN`, `$NC`, etc.), or use helper functions that include `-e`. | Discovered during Pi install — garbled output | CONFIRMED |
